# AI Tutor Chat Design

## Goal
Add a high-quality Tutor conversation page to the existing AI Tutor project, with backend-proxied multi-provider model access and a Pomodoro-driven study session flow.

## Product Flow
The current entry page remains mostly intact. It gains a clear action to enter today's learning or training session. Clicking that action switches the app into a Tutor workspace rather than a generic chat platform.

The workspace starts a learning session, shows a Pomodoro timer, lets the student talk with an AI Tutor, and keeps the selected provider/model visible. When the focus timer ends, the page reminds the student to rest or continue another round. Ending the session should leave room for a summary generated by the Tutor.

## Multi-Provider API
The frontend never receives API keys. It calls backend endpoints:

- `GET /api/llm/providers`: returns enabled provider/model metadata.
- `GET /api/llm/prompt-profiles`: returns built-in Tutor system prompt profiles.
- `POST /api/llm/chat`: accepts provider, model, prompt profile, optional system prompt override, messages, and Tutor context.

Providers are implemented behind backend adapters. First version prioritizes OpenAI-compatible providers:

- OpenAI
- DeepSeek
- Qwen
- Ollama

Anthropic and Gemini remain supported as provider metadata placeholders until native adapters are implemented.

## Prompt Profiles
The system prompt must not be fixed to only Socratic tutoring. Built-in profiles:

- `socratic`: guide through questions.
- `explain`: explain concepts clearly.
- `diagnose`: identify mistakes and misconceptions.
- `coach`: manage learning rhythm and Pomodoro reminders.
- `exam`: simulate exam-style practice.
- `custom`: accepts a backend-controlled or request-provided override.

The backend exposes prompt profile metadata and composes the selected system prompt for chat calls.

## Frontend UI
Entry page change is small: add a prominent study/training entry button in the existing `StartTraining` area.

Tutor workspace layout:

- Header/status bar: session state, remaining time, provider/model, exit action.
- Left rail: Pomodoro timer, session goal, prompt profile selector, provider/model selector.
- Main chat: message timeline, Tutor tags, quick teaching actions, input box.
- States: loading, provider not configured, empty conversation, timer break reminder.

Mobile layout stacks the controls above the chat and keeps the input reachable.

## Acceptance Criteria
- Backend exposes provider and prompt-profile metadata without leaking API keys.
- Backend can call OpenAI-compatible providers through a shared adapter.
- Chat requests include Tutor prompt profile behavior.
- Frontend entry page has a clear route into learning/training.
- Tutor workspace shows timer, provider/model, prompt profile, chat messages, quick actions, and input.
- Pomodoro state starts when entering the workspace and prompts the student at focus end.
- Build/import/unit validation passes in the staged workspace.
